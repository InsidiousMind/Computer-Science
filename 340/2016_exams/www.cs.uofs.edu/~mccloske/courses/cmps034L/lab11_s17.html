    public static void writeNums(int n)
{
   // Check for non-positive parameter and throw an exception in that case.
   if (n < 1) { throw new IllegalArgumentException(); }

   // Parameter is legal, so use normal pattern for a recursive functional method
   if (n == 1) {    // the base case
      System.out.print(n);
   }
   else {  // the recursive case
      writeNums(n-1);

Activity #3: Homegrown Recursive Programming Problems

As with your solutions to the exercises in Activity #2, place the methods that you develop here into the Lab11 class and, when you're finished, submit it.

  1. Develop a recursive method digitSum(int n) that, given a nonnegative integer via its parameter, returns the sum of the digits in the (decimal) numeral representing that number. Thus, for example, the call digitSum(6305) should yield 14, because that is the sum of 6, 3, 0, and 5.

    What is the base case? It could be n < 10 (meaning that n is represented by a one-digit numeral. Or it could be n==0 (meaning, conceptually, that n is represented by a zero-digit numeral?). Either one will work.

    As for the recursive case, note that the sum of the digits in 6305 (to use that example again) is the sum of the digits in 630, plus the one's digit, 5. So the key to solving (this instance of) the problem lies in obtaining 630 and 5 from 6305. But these are just the quotient and remainder resulting from dividing 6305 by ten! The same approach works for any integer.
