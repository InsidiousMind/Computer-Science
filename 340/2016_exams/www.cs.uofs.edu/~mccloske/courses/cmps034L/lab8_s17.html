Activity #4: In which you submit work!

  1. Download the TossableCoin2 and TossableCoin2App classes and load them into jGrasp (or whatever IDE you are using).

    Modify TossableCoin2App (and change its name to TossableCoin2LApp) so that, rather than tossing a coin ten times, it tosses the coin until either ten tosses have resulted in Heads or ten tosses have resulted in Tails. Note that objects that are instances of the TossableCoin2 class have observer methods (namely, headsCount() and tailsCount()) by which a client can determine how many times each of Heads and Tails has resulted from past coin tosses. Therefore, don't insert code in the client program for maintaining heads- and tails-counts! Rather, use the methods just mentioned to obtain those counts.

    Use this Submit/Review link to submit the TossableCoin2LApp class. (Do not submit TossableCoin2 at this time.)

  2. Now modify the TossableCoin2 class by replacing instance variable showingHeads by a new instance variable, faceShowing, of type char. The idea is that faceShowing's value should be either 'H' or 'T', respectively, depending upon whether the coin is showing Heads or Tails.

    Make whatever modifications are necessary to the source code in TossableCoin2 in order to make this work. (Obviously, any code in the class that refers to headsShowing must be changed, since that instance variable no longer plays any role in the class.)

    If your modifications are correct, you should be able to compile the modified class without errors being reported, and you should be able to run both the TossableCoin2App and TossableCoin2LApp applications (without making any changes to them!), and they should exhibit similar behavior to what they did before.
