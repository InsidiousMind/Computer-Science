Several of the self-checks involve evaluating one or more arithmetic expressions. Keep in mind the rules that govern the order in which operators are applied, which are explained in the Reges & Stepp textbook on pages 70-73. Unary operators (as in -13 or +9) have highest precedence, followed by the (binary) multiplicative operators (*, /, and %), followed by the (binary) additive operators (+ and -). Consecutive operators of the same precedence are applied from left to right.

Also keep in mind that if both operands of a division operator (/) are of type int, the result is also of type int.

Activity #3: Lengths

This exercise is to give you practice in formulating arithmetic expressions and in using methods. It is similar in nature to the BMICalculator3 application discussed in lecture. Note that the program's use of "global" variables (which is generally frowned upon) is something that you will be able to avoid in the near future, after you learn how to send arguments/parameters to a method and how to make a method return a value to its caller.

Given is an incomplete version of the Java application Lengths. Comments that say "CODE MISSING HERE!" indicate where code should be inserted in order to get the program working as intended. The program is intended to prompt the user to enter a positive integer representing a length, measured in inches. After reading the user's response, the program displays two alternative ways of expressing the same length.

For example, here's what the user-computer dialog should look like (with user input in boldface) if the user were to enter 147 in response to the prompt:
