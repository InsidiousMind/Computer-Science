  3. (Optional; If you have the time and inclination!)
    BJP4 Self-Check 5.16 (isPrime): Given is a flawed method isPrime() that (is intended to) determines whether or not the value passed to it is prime. You are to fix it.

    Hint: Recalling our discussion in class of the stringEquals() method (or whatever we called it) may help you to identify the mistake in the given code.

    Besides having a logic error, the given method has at least one more "deficiency": it is very inefficient. Consider that, as written, it will, if the value of n happens to be prime, evaluate the expression n % i for values of i from 2 all the way through n-1. But there is no need to have i go anywhere near that high, because if n is not prime, its smallest divisor (other than 1) must be far less than n-1.

    Use your mathematical intuition to come up with a number K (which will depend upon the value of n, of course) that is smaller than n-1 but is such that n is guaranteed to have a divisor in the range 2..K (assuming, of course, that n is not prime). Then adjust the source code of isPrime() so that, if it finds no divisors in the range 2..K, it returns true.

    Hint: Divisors of a number come in pairs. For example, (1,72), (2,36), (3,24), (4,18), (6,12), and (8,9) are the "divisor pairs" of 72.
