  1. BJP4 Self-Check 3.8 (parameterMysteryTouch): Show output produced by each call to the method touch().
    You must remember that the names of the formal parameters in a method have nothing to do with the names of variables that are used as actual parameters in a call to the method. It's the order in which the parameters are listed (in both the call and in the method heading) that determines the correspondence.

    For example (as depicted in the figure below), the call touch(ear, elbow) in main() results in the first actual parameter (ear, which has value "eye") being transmitted to the first formal parameter (elbow) and the value of the second actual parameter (elbow, which has value "head") being transmitted to the second formal parameter (ear). Thus, in response to this call, touch() will begin executing with its formal parameter elbow having value "eye" and its formal parameter ear having value "head", which are opposite of the values of the same-named variables in main(). 

                       +-----+             +------+
In main():     ear |"eye"|       elbow |"head"|
                   +-----+             +------+
                      |                   |
                      |                   |
                      |                   |
                      v                   v
                   +-----+            +------+
In touch():  elbow |"eye"|        ear |"head"|
                   +-----+            +------+

    Note that a statement such as Math.sqrt(count) evaluates to some value, but because that value is not being assigned to a variable (or being used in any other way, such as as a parameter in a call to a method), it has no lasting effect.
